# Descriptions — SUPER_ADMIN

- **Se connecter** : meme parcours que admin avec role super admin.
- **Second auth** : mot de passe secondaire obligatoire avant actions sensibles.
- **Impersonation** : prise de controle session admin pour support.
- **Valider/Rejeter admin_request** : workflow d'approbation des nouveaux admins.
- **Bloquer/Debloquer IP** : mitigation brute-force et abus.
- **Valider paiement cash** : marquer un paiement comme paid apres preuve.
- **Maintenance policy** : bascule mode maintenance/lecture seule.
